FastMCP vs MCP SDK — History

1

Birth of MCP (2023–2024)

  • Anthropic introduces MCP — structured communication between LLMs & external tools/data
  • Official Python SDK (mcp) released with: mcp.server · mcp.client · mcp.cli
  • Install: pip install mcp[cli] — becomes the reference implementation
2

Developer Friction & FastMCP 1.x (2024)

  • Problem: Raw SDK = verbose & boilerplate heavy — manual list_tools, call_tool, explicit schemas
  • Solution: FastMCP helper inside SDK (mcp.server.fastmcp) — decorator-driven, type-hint aware
  • FastMCP ≈ FastAPI for MCP, MCP SDK ≈ WSGI/Flask for Python web
  • Popular for teaching & prototyping
3

FastMCP Breaks Out (2025) — v2.x

  • FastMCP adoption grows rapidly; SDK maintainers split: SDK stays minimal & spec-focused
  • FastMCP 2.x released as standalone package — install: pip install fastmcp
  • Standalone, community-driven, fast-evolving; highly compatible with 1.x but may diverge
4

Where Things Stand (Late 2025)

MCP SDK (mcp)

  • Official, spec-compliant, stable
  • Install: pip install mcp[cli]

FastMCP 2.x (fastmcp)

  • Standalone, community-driven
  • Ideal for students, educators, rapid prototypes
  • Not official, but more ergonomic

Basic Setup — Demo Server

Setup steps:

  1. Install uv
  2. Create project folder: fastmcp-demo-server
  3. Open folder in VS Code, open terminal
  4. uv init .
  5. uv add fastmcp
  6. fastmcp version — verify install
  7. Create main.py with tools
  8. uv run fastmcp dev main.py → MCP Inspector
  9. uv run fastmcp run main.py → production run
  10. uv run fastmcp install claude-desktop main.py → adds to Claude Desktop

Demo server code (main.py):

import random
from fastmcp import FastMCP

mcp = FastMCP("Demo Server")

@mcp.tool
def roll_dice(n_dice: int = 1) -> list[int]:
    """Roll n_dice 6-sided dice."""
    return [random.randint(1, 6)
            for _ in range(n_dice)]

@mcp.tool
def add_numbers(a: float, b: float) -> float:
    """Add two numbers together."""
    return a + b

if __name__ == "__main__":
    mcp.run()

Expense Tracker — Full Server Code

from fastmcp import FastMCP
import os, sqlite3

DB_PATH = os.path.join(os.path.dirname(__file__), "expenses.db")
mcp = FastMCP("ExpenseTracker")

# ─── DB INIT ────────────────────────────────────────
def init_db():
    with sqlite3.connect(DB_PATH) as c:
        c.execute("""
            CREATE TABLE IF NOT EXISTS expenses(
                id INTEGER PRIMARY KEY AUTOINCREMENT,
                date TEXT NOT NULL,
                amount REAL NOT NULL,
                category TEXT NOT NULL,
                subcategory TEXT DEFAULT '',
                note TEXT DEFAULT ''
            )""")
init_db()

# ─── TOOL 1: Add Expense ────────────────────────────
@mcp.tool()
def add_expense(date, amount, category,
                  subcategory="", note=""):
    with sqlite3.connect(DB_PATH) as c:
        cur = c.execute(
            "INSERT INTO expenses(date,amount,category,subcategory,note) VALUES (?,?,?,?,?)",
            (date, amount, category, subcategory, note))
    return {"status": "ok", "id": cur.lastrowid}

# ─── TOOL 2: List Expenses (with date range) ────────
@mcp.tool()
def list_expenses(start_date, end_date):
    '''List expense entries within an inclusive date range.'''
    with sqlite3.connect(DB_PATH) as c:
        cur = c.execute("""
            SELECT id, date, amount, category, subcategory, note
            FROM expenses
            WHERE date BETWEEN ? AND ?
            ORDER BY id ASC""", (start_date, end_date))
        cols = [d[0] for d in cur.description]
        return [dict(zip(cols, r)) for r in cur.fetchall()]

# ─── TOOL 3: Summarize (by category, optional filter) 
@mcp.tool()
def summarize(start_date, end_date, category=None):
    '''Summarize expenses by category within date range.'''
    with sqlite3.connect(DB_PATH) as c:
        query = """SELECT category, SUM(amount) AS total_amount
                   FROM expenses WHERE date BETWEEN ? AND ?"""
        params = [start_date, end_date]
        if category:
            query += " AND category = ?"
            params.append(category)
        query += " GROUP BY category ORDER BY category ASC"
        cur = c.execute(query, params)
        cols = [d[0] for d in cur.description]
        return [dict(zip(cols, r)) for r in cur.fetchall()]

# ─── RESOURCE: Categories JSON ───────────────────────
CATEGORIES_PATH = os.path.join(os.path.dirname(__file__), "categories.json")

@mcp.resource("expense://categories", mime_type="application/json")
def categories():
    # Read fresh each time so you can edit without restarting
    with open(CATEGORIES_PATH, "r", encoding="utf-8") as f:
        return f.read()

if __name__ == "__main__":
    mcp.run()

FastMCP × FastAPI Integration

Design Philosophy

FastMCP is to MCP as FastAPI is to WSGI. They share the same decorator-driven, type-hint-aware design. A FastAPI app can be wrapped as an MCP server in just a few lines.

Real-world Scenario

Companies already have products (website + FastAPI backend). MCP connects those APIs to Claude Desktop / ChatGPT clients — without rewriting anything.

server.py — Convert FastAPI → MCP (13 lines)
# server.py — mcp_server.py
from fastmcp import FastMCP
from main import app  # Import your FastAPI app

# Convert FastAPI app to MCP server — one line!
mcp = FastMCP.from_fastapi(
    app=app,
    name="Expense Tracker server",
)

if __name__ == "__main__":
    mcp.run()

Run dev inspector: uv run fastmcp dev server.py

Decorator Use
@mcp.tool() Expose function as MCP tool
@mcp.resource("uri") Expose data as MCP resource
@mcp.prompt() Expose prompt template
