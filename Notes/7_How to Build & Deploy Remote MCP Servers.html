Local vs Remote MCP — Concepts

📦 Local MCP Server
  • Runs on your machine
  • Communication via STDIO
  • Fast — no network latency
  • Single client only
  • Best for personal / dev use
🌐 Remote MCP Server
  • Hosted on the internet
  • Communication over network
  • Slower (network overhead)
  • Multiple clients can connect
  • Powerful — enterprise-grade

Key Insight

Enterprise MCP servers → Remote servers. Remote servers are more powerful because they can serve many clients simultaneously and run on dedicated infrastructure. The trade-off is network latency vs capability and reach.

03

Simple Remote Server — Calculator

Structure

A FastMCP server with 2 tools and 1 resource, running over HTTP on port 8000 with host 0.0.0.0 for remote access.

main.py — Simple Calculator Server Python 
from fastmcp import FastMCP
import random
import json

# Create the FastMCP server instance
mcp = FastMCP("Simple Calculator Server")

# Tool 1: Add two numbers
@mcp.tool
def add(a: int, b: int) -> int:
    """Add two numbers together.

    Args:
        a: First number
        b: Second number

    Returns:
        The sum of a and b
    """
    return a + b

# Tool 2: Generate a random number
@mcp.tool
def random_number(min_val: int = 1, max_val: int = 100) -> int:
    """Generate a random number within a range.

    Args:
        min_val: Minimum value (default: 1)
        max_val: Maximum value (default: 100)

    Returns:
        A random integer between min_val and max_val
    """
    return random.randint(min_val, max_val)

# Resource: Server information
@mcp.resource("info://server")
def server_info() -> str:
    """Get information about this server."""
    info = {
        "name": "Simple Calculator Server",
        "version": "1.0.0",
        "description": "A basic MCP server with math tools",
        "tools": ["add", "random_number"],
        "author": "Your Name"
    }
    return json.dumps(info, indent=2)

# Start the server
if __name__ == "__main__":
    mcp.run(transport="http", host="0.0.0.0", port=8000)
💡

Key difference from local: transport="http" + host="0.0.0.0" makes it a remote server. Local servers use default STDIO transport.

Expense Tracker MCP Server

Overview

A production-style MCP server with an SQLite database backend. Exposes 3 tools: add_expense list_expenses summarize and 1 resource: expense:///categories

main.py — Expense Tracker (initial version) Python 
from fastmcp import FastMCP
import os
import sqlite3

DB_PATH = os.path.join(os.path.dirname(__file__), "expenses.db")
CATEGORIES_PATH = os.path.join(os.path.dirname(__file__), "categories.json")

mcp = FastMCP("ExpenseTracker")

def init_db():
    with sqlite3.connect(DB_PATH) as c:
        c.execute("""
            CREATE TABLE IF NOT EXISTS expenses(
                id INTEGER PRIMARY KEY AUTOINCREMENT,
                date TEXT NOT NULL,
                amount REAL NOT NULL,
                category TEXT NOT NULL,
                subcategory TEXT DEFAULT '',
                note TEXT DEFAULT ''
            )
        """)

Database Schema

  • id — INTEGER PRIMARY KEY AUTOINCREMENT
  • date — TEXT NOT NULL
  • amount — REAL NOT NULL
  • category — TEXT NOT NULL
  • subcategory — TEXT DEFAULT ''
  • note — TEXT DEFAULT ''

Default Categories (categories.json)

Food & Dining, Transportation, Shopping, Entertainment, and more — stored as JSON alongside the server code.

Fixing Issues in the Remote Server

Problem 1: Read-Only Database

FastMCP Cloud runs in a read-only filesystem. The expenses.db file cannot be written to at the code directory path.


Fix: Use tempfile.gettempdir() to store the DB in a writable temp directory.

Problem 2: Sync / Async Conflict

FastMCP remote servers run in an async environment. Using synchronous sqlite3 causes blocking issues.


Fix: Replace sqlite3 with aiosqlite and make all DB functions async.

Summary of All Changes

  • Import aiosqlite instead of sqlite3
  • Import tempfile and set DB_PATH to tempfile.gettempdir()
  • Add async to all database functions
  • Replace sqlite3.connect → aiosqlite.connect
  • Add async with context manager for connections

Proxy Server — Connecting Claude Desktop to Remote MCP

Why a Proxy?

Claude Desktop connects to MCP servers via STDIO. Remote MCP servers speak HTTP. A tiny local proxy bridges the gap — it runs locally via STDIO and forwards all requests to the remote FastMCP Cloud server over HTTP.

Proxy Architecture
Claude Desktop
(CD)
⟷ STDIO
Proxy Server
(local MCP)
HTTP ⟷
Remote MCP
(FastMCP Cloud)
FastMCP Cloud uses Streamable HTTP by default — use the /mcp URL directly
proxy-server/main.py Python 
from fastmcp import FastMCP

# Create a proxy to your remote FastMCP Cloud server
# FastMCP Cloud uses Streamable HTTP (default), so just use the /mcp URL
mcp = FastMCP.as_proxy(
    "https://lovely-silver-newt.fastmcp.app/mcp",  # Standard FastMCP Cloud URL
    name="Nitish Server Proxy"
)

if __name__ == "__main__":
    # This runs via STDIO, which Claude Desktop can connect to
    mcp.run()

How to Add to Claude Desktop

Configure claude_desktop_config.json to run this proxy script. Claude Desktop launches it as a subprocess via STDIO.


Manual connector

FastMCP.as_proxy()

One-line method to wrap any remote FastMCP HTTP server as a local STDIO-compatible proxy. No boilerplate needed.
